## Configure Next.js

Wrap your main Next.js config with the `withWorkflow()` function:

```ts filename="next.config.ts" {1,4-6,8}
import { withWorkflow } from '@vercel/workflow-next';
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  // … rest of your Next.js config
};

export default withWorkflow(nextConfig);
```

Wrapping your config with `withWorkflow()` will:

- Add a Turbopack/webpack loader that compiles files containing `"use workflow"`/`"use step"`
- Build the workflow and step bundles during `next build` via the Workflow CLI
- Enable the embedded transport in dev for local testing

## Create your first workflow

Create a new file in the `workflows` directory:

```ts filename="workflows/hello-world.ts"
export async function helloWorkflow(word: string) {
  "use workflow";

  const greeting = await createGreeting(word);
}

async function createGreeting(word: string) {
  "use step";

  // This runs with full Node.js access
  console.log(`Running greeting step with word: ${word}`)
  return `Hello ${word}!`;
}
```

<Callout type="info">
Workflow functions orchestrate execution, while step functions perform the actual work. Learn more in [Concepts](./concepts).
</Callout>

## Invoke your workflow

```ts filename="app/api/hello/route.ts"
import { helloWorkflow } from '@/workflows/hello-world';
import { NextResponse } from 'next/server';

export async function GET(request: Request) {
  // Start the workflow (non-blocking)
  await helloWorkflow("world");

  return NextResponse.json({
    message: `Workflow started`
  });
}
```

<Callout>
Workflows can be triggered from API routes, Server Actions, or any server-side code.
</Callout>

<Callout type="info">
Workflows execute asynchronously and don't block your application. The SDK handles queuing, execution, and retry logic automatically.
</Callout>
