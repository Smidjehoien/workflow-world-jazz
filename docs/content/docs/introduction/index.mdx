# Workflow SDK

The Workflow SDK is a TypeScript framework to easily build durable, reliable, and observable applications and AI agents.

Workflows are a powerful primitive to power anything from data pipelines, AI agents, media processing jobs, billing and subscriptions, CI/CD flows, and everything in between.

Check out the [examples](/examples) for more inspiration.

---

## Why use Workflow SDK?

Building reliable long-running processes is hard. You'll need to handle failures, manage retries, persist state, wire up queues, coordinate distributed operations, and deal with timeouts. Traditional approaches require complex infrastructure and careful state management.

The Workflow SDK eliminates these complexities by providing durable functions that automatically persist progress, retry failures, and track progress for observability. This allows developers to write resilient workflows using **familiar async/await patterns**, without worrying about infrastructure details.

For example, here's a simple order processing workflow powered by workflow.

```typescript title="workflow.ts"
import { sleep, createWebhook } from "@vercel/workflow"

export async function fulfillOrder(orderId: string) {
  "use workflow";

  // In a workflow, steps are queued in the background while
  // the workflow function suspends
  const order = await deliverOrder(orderId);

  await sleep("2 days"); // Workflows can suspend for a long time between steps

  const webhook = createWebhook(); // They can even react to external events!
  await sendEmail(`
    Dear ${order.buyer}, please confirm your order was received: ${webhook.url}
  `)
  await webhook; // Workflow suspends until the email link is clicked

  console.log("Delivery complete!")
}

async function deliverOrder(orderId: string) {
  "use step";

  // Steps are run on separate serverless functions and
  // represents a single unit of work

  // Uncaught errors (ex. network errors) are automatically retried
  if (Math.random() < 0.5) {
    throw new Error("Error starting delivery. Retrying...");
  }

  console.log("Delivering order", orderId);

  return { buyer: "Evil Rabbit" };
}
```

The key takeaway is that we can easily define workflows through workflow functions and step functions with both `"use workflow"` and `"use step"` directives. You can learn more about both in [Concepts](/docs/foundations/concepts).
