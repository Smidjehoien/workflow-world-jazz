# Runtime Issues

Troubleshoot common problems in development when executing workflows.

## Context Function Errors

Functions like `getStepContext`, and `sleep` from the Workflow SDK Core package depend on the execution context (i.e. *where* the function is executed).

### Accessing Context

Workflow SDK Core provides the following functions for accessing context:

* [`getWorkflowContext`](/docs/api-reference/get-workflow-context)
* [`getStepContext`](/docs/api-reference/get-step-context).

Trying to call `getStepContext` inside a workflow function isn't possible since no current executing step exists.

```ts
import { getWorkflowContext, getStepContext } from "@vercel/workflow-core"

// getWorkflowContext() - available in workflow and step functions
export async function myWorkflow() {
    "use workflow";

    const ctx = getWorkflowContext();
    console.log(ctx.workflowRunId) // Valid

    await testStep();

    const stepCtx = getStepContext() // Invalid, outside execution context
}

async function testStep() {
    "use step"

    const ctx = getStepContext()
    console.log(ctx.stepId)        // Valid, within scope of a step function
}
```

## Violating Determinism in Workflows

Workflow functions must contain deterministic behavior in order for the workflow to precisely resume state after some interruption (e.g. through errors, or suspending through `sleep`).

Using non-deterministic functions inside workflow functions directly can have un-predictable behavior.

If non-deterministic behavior is intended, it's important to move that to step functions.

```typescript lineNumbers
// Bad - in workflow
export async function myWorkflow() {
  "use workflow";
  const id = Math.random(); // Seeded per workflow
  const now = Date.now();   // Fixed (then incremented)
}
 
// Good - in step
async function generateId() {
  "use step";
  return Math.random(); // Unseeded, normal behavior
}
 
export async function myWorkflow() {
  "use workflow";
  const id = await generateId();
}
```

From this example:

* `Math.random()` is seeded per workflow in order to ensure determinism.
* `Date.now()` is fixed then incremented throughout the lifetime of the workflow after completed steps.
