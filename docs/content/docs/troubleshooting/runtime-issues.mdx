# Runtime Issues

Troubleshoot common problems in development when executing workflows.

## Context Function Errors

Functions like `getContext`, and `sleep` from the [Workflow SDK Core](/docs/workflow-sdk-core) package depend on the execution context (i.e. *where* the function is executed). This means that their uses are limited depending on if they are called inside a workflow function or a step function.

### Accessing Context

Workflow SDK Core provides the function `getContext`. This function can be used to return the metadata of a workflow or a step depending on where it's executed.

Some fields of `getContext` will not be available when called in a workflow function if it doesn't exist. For example, accessing the `stepId` or `stepStartedId` fields inside a workflow function isn't possible since no step is currently being executed.

```ts
// getContext() - available in workflow and step functions
export async function myWorkflow() {
    "use workflow";

    const ctx = getContext();
    console.log(ctx.workflowRunId) // Valid

    await testStep();
    console.log(ctx.stepId)        // Invalid, outside execution context
}

async function testStep() {
    "use step"

    const ctx = getContext()
    console.log(ctx.stepId)        // Valid, within scope of a step function
}
```

You can learn more about accessing context in the [`getContext`](/docs/workflow-sdk-core/get-context) section.

### Sleeping a Workflow

## Violating Determinism in Workflows

Workflow functions must contain deterministic behavior in order for the workflow to precisely resume state after some interruption (e.g. through errors, or suspending through `sleep`).

Using non-deterministic functions inside workflow functions directly can have un-predictable behavior.

If non-deterministic behavior is intended, it's important to move that to step functions.

```typescript lineNumbers
// Bad - in workflow
export async function myWorkflow() {
  "use workflow";
  const id = Math.random(); // Seeded per workflow
  const now = Date.now();   // Fixed (then incremented)
}
 
// Good - in step
async function generateId() {
  "use step";
  return Math.random(); // Unseeded, normal behavior
}
 
export async function myWorkflow() {
  "use workflow";
  const id = await generateId();
}
```

From this example:

* `Math.random()` is seeded per workflow in order to ensure determinism.
* `Date.now()` is fixed then incremented throughout the lifetime of the workflow after completed steps.
