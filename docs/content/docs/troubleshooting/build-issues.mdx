# Build Issues

Investigate common problems when creating workflows.

## Workflows Not Detected

When workflows aren't being invoked or built during local development or in a production environment, verify that you are using the `"use workflow"` and `"use step"` directives appropriately.

For workflow functions:

```ts lineNumbers
// Correct - includes directive
export async function orderWorkflow(orderId: string) {
  "use workflow"; // [!code highlight]
  // workflow logic
}

// Wrong - missing directive
export async function orderWorkflow(orderId: string) {
  // workflow logic
}
```

For step functions:

```typescript lineNumbers
// Correct - includes directive
export async function emailOrder(orderId: string, email: string) {
  "use step"; // [!code highlight]
  // step logic
}

// Wrong - missing directive
export async function emailOrder(orderId: string, email: string) {
  // step logic
}
```

Make sure you've wrapped the Next.js configuration with the `withWorkflow` plugin:

```ts lineNumbers title="next.config.ts"
import { withWorkflow } from '@vercel/workflow-next'; // [!code highlight]

const nextConfig = {
  // your config
};

export default withWorkflow(nextConfig); // [!code highlight]
```

## External package errors

Sometimes, some packages require special configuration to work with workflows. You should add these packages to `serverExternalPackages` in your `next.config.ts`:

```ts lineNumbers=3 title="next.config.ts"
const nextConfig = {
  serverExternalPackages: [
    '@node-rs/xxhash',
    'sharp',
    'canvas'
  ]
};
```
