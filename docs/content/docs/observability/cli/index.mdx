# Workflow CLI

The Workflow CLI (`@vercel/workflow-cli`) is a command-line tool for building, inspecting, and managing workflows. It provides both `workflow` and `wf` (shorthand) commands.

## Installation

```bash
npm install @vercel/workflow-cli
```

## Quick Start

### Inspecting Local Workflows

By default, the CLI and Web UI inspect your local development environment:

```bash
# Get started by listing recent workflow runs
wf i runs

# Launch Web UI for visual exploration, running locally on port 3456
wf i runs --web

# See all available commands
wf i --help
```

See the [API Reference: CLI](/docs/api-reference/cli) for detailed command documentation.

### Inspecting Vercel Deployments

To inspect workflows running on Vercel, add the `--backend vercel` flag:

```bash
# Inspect production workflows
wf i runs --backend vercel

# Launch Web UI for production
wf i runs --web --backend vercel
```

## Vercel CLI Setup

To inspect a Vercel deployment, you need to be authenticated and [link your Vercel project](https://vercel.com/docs/cli/project-linking):

1. **Login to the Vercel CLI:**

   ```bash
   npx vercel login
   ```

2. **Link your Vercel project:**

   ```bash
   npx vercel link
   ```

3. **Inspect workflows:**

   ```bash
   wf i runs --backend vercel
   ```

The CLI will automatically use your Vercel authentication and project configuration.

Additionally, the `--env` flag is available to specify a different environment.

```bash
# Inspect preview environment
wf i runs --backend vercel --env preview

# Inspect development environment  
wf i runs --backend vercel --env development
```

## Environment Variables

Configuration can also be provided via environment variables:

**Vercel World:**

```bash
export WORKFLOW_TARGET_WORLD=vercel
export WORKFLOW_VERCEL_AUTH_TOKEN=<your-token>
export WORKFLOW_VERCEL_ENV=production
export WORKFLOW_VERCEL_PROJECT_ID=<project-id>
export WORKFLOW_VERCEL_TEAM_ID=<team-id>
```

**Embedded World:**

```bash
export WORKFLOW_TARGET_WORLD=embedded
export WORKFLOW_EMBEDDED_DATA_DIR=./.workflow-data
```

**Web UI:**

```bash
export PORT=3456
```

## Configuration

The CLI looks for workflow files in the `./workflows/` directory by default:

```
workflows/
├── user-signup.ts
├── email-campaign.ts
└── utils/
```

## Examples

### Inspect Local Workflows

```bash
# View recent runs
npx wf i runs

# View specific run
npx wf i runs <run-id>

# View steps in a run
npx wf i steps <run-id>

# Launch Web UI
npx wf i runs --web
```

### Inspect Vercel Production

```bash
# View production runs
npx wf i runs --backend vercel

# View production runs in Web UI
npx wf i runs --web --backend vercel

# View preview environment
npx wf i runs --backend vercel --env preview
```

### Build Workflows

```bash
# Build with default target
npx workflow build

# Build for Vercel static
npx workflow build --target vercel-static

# Build with Vercel Build Output API
npx workflow build --target vercel-build-output-api
```
