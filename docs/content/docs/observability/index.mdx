# Observability

Workflow DevKit provides powerful tools to inspect, monitor, and debug your workflows through the CLI.

The CLI is a command-line tool that provides a set of commands to inspect, monitor, and debug your workflows.

---

## Installation

```bash
npm install @vercel/workflow-cli
```

---

## Quick Start

Get started inspecting your local workflows:

```bash
# Get started by listing recent workflow runs
npx wf i runs

# Launch Web UI for visual exploration, running locally on port 3456
npx wf i runs --web

# See all available commands
npx wf i --help
```

See the [CLI Reference](/docs/cli-reference) for detailed command documentation.

## Backends

### Local Development

By default, the CLI and Web UI inspect your local development environment:

```bash
npx wf i runs
```

### Vercel Deployments

Inspect workflows running on Vercel by adding the `--backend vercel` flag:

```bash
# Inspect production workflows
npx wf i runs --backend vercel

# Launch Web UI for production
npx wf i runs --web --backend vercel
```

<Callout>
Requires [Vercel CLI authentication](https://vercel.com/docs/cli/project-linking) and project linking.
</Callout>

#### Linking your Vercel Project

1. **Login to the Vercel CLI:**

   ```bash
   npx vercel login
   ```

2. **Link your Vercel project:**

   ```bash
   npx vercel link
   ```

3. **Inspect workflows:**

   ```bash
   npx wf i runs --backend vercel
   ```

The CLI will automatically use your Vercel authentication and project configuration.

Additionally, the `--env` flag is available to specify a different environment.

```bash
# Inspect preview environment
npx wf i runs --backend vercel --env preview

# Inspect development environment
npx wf i runs --backend vercel --env development
```

## What You Can Inspect

### Runs

View all workflow executions, their status, and timing information.

```bash
npx wf i runs
```

### Steps

See individual steps within a workflow run, including inputs, outputs, and execution time.

```bash
npx wf i steps <run-id>
```

### Streams

Monitor streaming data and events within your workflows.

```bash
npx wf i streams -r <run-id>
```

---

## Web UI

The CLI also has a web UI for inspecting your workflows with the `--web` flag.

```bash
# Local workflows
npx wf i runs --web

# Production workflows
npx wf i runs --web --backend vercel
```

The Web UI provides:

- Timeline views of workflow execution
- Step-by-step debugging
- Stream visualization
- Error tracking

## Examples

### Inspect Local Workflows

```bash
# View recent runs
npx wf i runs

# View specific run
npx wf i runs <run-id>

# View steps in a run
npx wf i steps <run-id>

# Launch Web UI
npx wf i runs --web
```

### Inspect Vercel Production

```bash
# View production runs
npx wf i runs --backend vercel

# View production runs in Web UI
npx wf i runs --web --backend vercel

# View preview environment
npx wf i runs --backend vercel --env preview
```

To learn more about the CLI, see the [CLI Reference](/docs/cli-reference).
