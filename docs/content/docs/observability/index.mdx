# Observability

Workflow provides powerful tools to inspect, monitor, and debug your workflows through the CLI and Web UI.

## Installing the CLI

Install the CLI globally:

```bash
npm install -g @vercel/workflow-cli@latest
```

## Quick Start

### Inspecting Local Workflows

By default, the CLI and Web UI inspect your local development environment:

```bash
# Get started by listing recent workflow runs
wf i runs

# Launch Web UI for visual exploration, running locally on port 3456
wf i runs --web

# See all available commands
wf i --help
```

See the [API Reference: CLI](/docs/api-reference/cli) for detailed command documentation.

### Inspecting Vercel Deployments

To inspect workflows running on Vercel, add the `--backend vercel` flag:

```bash
# Inspect production workflows
wf i runs --backend vercel

# Launch Web UI for production
wf i runs --web --backend vercel
```

For this to work, you need to be authenticated and have your project linked. See the Vercel CLI Setup section below.

## Vercel CLI Setup

To inspect a Vercel deployment, you need to be authenticated and [link your Vercel project](https://vercel.com/docs/cli/project-linking):

1. **Login to the Vercel CLI:**

   ```bash
   npx vercel login
   ```

2. **Link your Vercel project:**

   ```bash
   npx vercel link
   ```

3. **Inspect workflows:**

   ```bash
   wf i runs --backend vercel
   ```

The CLI will automatically use your Vercel authentication and project configuration.

Additionally, the `--env` flag is available to specify a different environment.

```bash
# Inspect preview environment
wf i runs --backend vercel --env preview

# Inspect development environment  
wf i runs --backend vercel --env development
```

## Custom Backends

The observability tools are compatible with custom world implementations and third-party backends. Follow the configuration instructions for your appropriate backend.
