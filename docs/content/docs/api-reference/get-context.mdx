# `getContext`

Returns additional metadata available in the current workflow function or step function. This function's outputs are execution dependent (i.e. whether it's called inside a workflow function or step function).

You may want to use this function when you need to:

* Log workflow run IDs
* Track retry attempts in error handling
* Access timing information of a step and execution metadata

## Import

```typescript
import { getContext } from "@vercel/workflow-core"
```

## API Signature

### Parameters

The `getContext` function takes no parameters.

### Returns

<TypeTable
  type={{
    workflowRunId: {
      type: "string",
      required: true,
      description: 'Unique identifier for the workflow run.',
    },
    workflowStartedAt: {
      type: "Date",
      required: true,
      description: 'Timestamp when the workflow run started.',
    },
    stepId: {
      type: "string",
      required: true,
      description: 'Unique identifier for the currently executing step. Only available inside a step function.'
    },
    stepStartedAt: {
      type: "Date",
      required: true,
      description: 'Timestamp when the current step started. Only available inside a step function.',
    },
    attempt: {
      type: "number",
      required: true,
      description: 'The number of times the current step has been executed. This value will increase with each retry. Only available inside a step function',
    },
    url: {
      type: "string",
      required: true,
      description: 'The URL where the workflow can be triggered.',
    }
  }}
/>

<Callout type="warn">
`stepId`, `stepStartedAt`, and `attempt` are properties that are only available inside step functions.
</Callout>

## Examples

Here's a simple example of how you can access the `workflowRunId` in a workflow, and the `stepId` inside a step function.

```typescript lineNumbers
import { getContext } from "@vercel/workflow-core" // [!code highlight]

async function testWorkflow() {
    "use workflow"

    const ctx = getContext() // [!code highlight]
    console.log(ctx.workflowRunId)

    await logStepId()
}

async function logStepId() {
    "use step"

    const ctx = getContext() // [!code highlight]
    console.log(ctx.stepId)
}
```
