# `getStepContext`

Returns additional metadata available in the current workflow function and step function.

You may want to use this function when you need to:

* Log workflow run IDs
* Track retry attempts in error handling
* Access timing information of a step and execution metadata

<Callout type="warn">
This function can only be called inside a step function.
</Callout>

## Import

```typescript
import { getStepContext } from "@vercel/workflow-core"
```

## API Signature

### Parameters

The `getStepContext` function takes no parameters.

### Returns

<TypeTable
  type={{
      workflowRunId: {
      type: "string",
      required: true,
      description: 'Unique identifier for the workflow run.',
    },
    workflowStartedAt: {
      type: "Date",
      required: true,
      description: 'Timestamp when the workflow run started.',
    },
    url: {
      type: "string",
      required: true,
      description: 'The URL where the workflow can be triggered.',
    },
    stepId: {
      type: "string",
      required: true,
      description: 'Unique identifier for the currently executing step. Only available inside a step function.'
    },
    stepStartedAt: {
      type: "Date",
      required: true,
      description: 'Timestamp when the current step started. Only available inside a step function.',
    },
    attempt: {
      type: "number",
      required: true,
      description: 'The number of times the current step has been executed. This value will increase with each retry. Only available inside a step function',
    },
  }}
/>

<Callout>
`getStepContext` extends [`getWorkflowContext`](/docs/api-reference/get-workflow-context) to add additional metadata of the current step run execution.
</Callout>

## Examples

Here's a simple example of how you can access the `workflowRunId`, and `stepId` inside a step function.

```typescript lineNumbers
import { getStepContext } from "@vercel/workflow-core" // [!code highlight]

async function testWorkflow() {
    "use workflow"

    await logStepId()
}

async function logStepId() {
    "use step"

    const ctx = getStepContext() // [!code highlight]
    console.log(ctx.workflowRunId) // [!code highlight]
    console.log(ctx.stepId) // [!code highlight]
}
```
