# `FatalError`

When a `FatalError` is thrown in a step, it indicates that the workflow should not retry a step, marking it as failure.

You should use this when you don't want a specific step to retry.

## Import

```typescript
import { FatalError } from "@vercel/workflow-core"
```

## Examples

Here's a simple example of how you can use `FatalError` inside your workflow to avoid retrying steps.

```typescript lineNumbers
import { FatalError } from "@vercel/workflow-core" // [!code highlight]

async function failableWorkflow() {
    "use workflow"

    await failStep();
}

async function failStep() {
    "use step"

    throw new FatalError("Failable!")
}
```
