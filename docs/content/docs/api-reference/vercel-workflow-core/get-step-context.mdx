# `getStepContext`

Returns metadata available in the current step function.

You may want to use this function when you need to:

* Track retry attempts in error handling
* Access timing information of a step and execution metadata

<Callout type="warn">
This function can only be called inside a step function.
</Callout>

## Import

```typescript
import { getStepContext } from "@vercel/workflow-core"
```

## Examples

Here's a simple example of how you can access the `stepId` inside a step function.

```typescript lineNumbers
import { getStepContext } from "@vercel/workflow-core" // [!code highlight]

async function testWorkflow() {
    "use workflow"
    await logStepId()
}

async function logStepId() {
    "use step"

    const ctx = getStepContext() // [!code highlight]
    console.log(ctx.stepId) // Grab the current step ID // [!code highlight]
}
```
