# `sleep()`

Suspends a workflow for a specified duration or until an end date without consuming any resources. Once the duration or end date has passed, the workflow will resume execution.

This is useful when you want to resume a workflow after some duration or date.

<Callout>
`sleep` is a *special* type of step function provided and should be called directly inside workflow functions.
</Callout>

## Import

```typescript
import { sleep } from "@vercel/workflow-core"
```

## Examples

Here's a simple example of how you can use `sleep` inside your workflow.

```typescript lineNumbers
import { sleep } from "@vercel/workflow-core" // [!code highlight]

async function testWorkflow() {
    "use workflow"

    // Sleeping with a duration
    await sleep("1d") // [!code highlight]
    // Workflow resumes after a day

    // Sleeping until an end date
    await sleep(new Date(Date.now() + 10_000)) // [!code highlight]
    // Workflow resumes after 10 seconds
}
```

We call `sleep("1d")` which suspends the workflow for 1 day. While the workflow is suspended, it doesn't use any resources. After a day, the workflow will resume execution at exactly where it left off.
