# `@vercel/workflow-core`

Core workflow primitives including steps, context management, streaming, webhooks, and error handling.

## Installation

<BaseUrl>
  {(baseUrl) => (
    <Tabs items={['pnpm', 'npm', 'yarn']}>
      <Tab value="pnpm">
        <CodeBlock>
            pnpm i {baseUrl}/vercel-workflow-core.tgz
        </CodeBlock>
      </Tab>
      <Tab value="npm">
        <CodeBlock>
            npm i {baseUrl}/vercel-workflow-core.tgz
        </CodeBlock>
      </Tab>
      <Tab value="yarn">
        <CodeBlock>
            yarn add {baseUrl}/vercel-workflow-core.tgz
        </CodeBlock>
      </Tab>
    </Tabs>
  )}
</BaseUrl>

## Core Functions

Workflow SDK Core contains the following core functions you can use inside your workflow functions:

<Cards>
    <Card href="/docs/api-reference/vercel-workflow-core/get-workflow-metadata" title="getWorkflowMetadata()">
        A function that returns context about the current workflow execution.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/get-step-metadata" title="getStepMetadata()">
        A function that returns context about the current step execution.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/sleep" title="sleep()">
        Sleeping workflows for a specified duration. Deterministic and replay-safe.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/fetch" title="fetch()">
        Make HTTP requests from within a workflow with automatic retry semantics.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/create-hook" title="createHook()">
        Create a low-level hook to receive arbitrary payloads from external systems.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/define-hook" title="defineHook()">
        Type-safe hook helper for consistent payload types.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/create-webhook" title="createWebhook()">
        Create a webhook that suspends the workflow until an HTTP request is received.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/get-workflow-writable-stream" title="getWorkflowWritableStream()">
        Access the current workflow run's default stream.
    </Card>
</Cards>

## Runtime Functions

Workflow SDK Core also provides runtime functions that are used outside of workflow and step functions. These are accessed from the runtime entrypoint (e.g. where `start(workflowFn)` is called):

<Cards>
    <Card href="/docs/api-reference/vercel-workflow-core/start" title="start()">
        Start/enqueue a new workflow run.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/resume-hook" title="resumeHook()">
        Resume a workflow by sending a payload to a hook.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/resume-webhook" title="resumeWebhook()">
        Resume a workflow by sending a `Request` to a webhook.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/get-workflow-return-value" title="getWorkflowReturnValue()">
        Get the final return value of a completed workflow run.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/get-workflow-run" title="getWorkflowRun()">
        Get workflow run status and metadata without waiting for completion.
    </Card>
</Cards>

## Error Classes

Workflow SDK Core includes error classes that can be thrown in a workflow or step to change the error exit strategy of a workflow.

<Cards>
    <Card href="/docs/api-reference/vercel-workflow-core/fatal-error" title="FatalError()">
        When thrown, marks a step as failed and the step is not retried.
    </Card>
    <Card href="/docs/api-reference/vercel-workflow-core/retryable-error" title="RetryableError()">
        When thrown, marks a step as retryable with an optional parameter.
    </Card>
</Cards>
