# Getting Started

<Callout type="warn">
Currently, Workflow SDK works best with Next.js. We're actively working on improving support and docs to use workflow by itself or with other frameworks.
</Callout>

This guide will walk through setting up your first workflow in a Next.js app. Along the way, you'll learn more about the concepts that are fundamental to using the SDK in your own projects.

---

## Create Your Next.js Project

Start by creating a new Next.js project. This command will create a new directory named `my-workflow-app` and setup a Next.js project inside it.

```bash
pnpm create next-app@latest my-workflow-app
```

Enter the newly made directory:

```bash
cd my-workflow-app
```

### Install Dependencies

Install `@vercel/workflow-core`, and `@vercel/workflow-next`, which are the packages needed to get started with the Workflow SDK.

### Configure Next.js

For workflows to work, you'll have to wrap your `next.config.ts` with `withWorkflow()`. This enables the directives `"use workflow"` and `"use step"` to use workflows in both local development and when deployed on Vercel.

```typescript title="next.config.ts" lineNumbers
import { withWorkflow } from '@vercel/workflow-next'; // [!code highlight]
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  // … rest of your Next.js config
};
 
export default withWorkflow(nextConfig); // [!code highlight]
```

## Create Your First Workflow

To create your first workflow, create a new `workflows` directory inside your project root (if using `src` folder then it would be `src/workflows`).

Create a new file inside the `workflows` directory and add the following code:

```typescript title="workflows/user-signup.ts" lineNumbers
import { sleep } from "@vercel/workflow-core";
 
export async function userSignupWorkflow(email: string) {
 "use workflow"; // [!code highlight]
 
 const user = await createUser(email);
 await sendWelcomeEmail(user.id);
 
 await sleep("30s"); // Pause for 30s - doesn't consume any resources
 await sendOnboardingEmail(user.id);
 
 return { userId: user.id, status: "onboarded" };
}
 
async function createUser(email: string) {
 "use step"; // [!code highlight]
 
 // Full Node.js access - database calls, APIs, etc.
 console.log(`Creating user with email: ${email}`);
 return { id: crypto.randomUUID(), email };
}
 
async function sendWelcomeEmail(userId: string) {
 "use step"; // [!code highlight]
 
 console.log(`Sending welcome email to user: ${userId}`);
 
 if (Math.random() < 0.3) {
  // By default, steps will be retried 3 times for unhandled errors
  throw new Error("Retryable!");
 }
}
 
async function sendOnboardingEmail(userId: string) {
 "use step"; // [!code highlight]
 
 console.log(`Sending onboarding email to user: ${userId}`);
 // Email API call here
}
```

Taking a look at this code:

* We define a **workflow** function with the directive `"use workflow"`. Think of the workflow function as the *orchestrator* of individual **steps**.
* Business logic lives inside **steps**, like the `createUser` step. When a step is invoked inside a **workflow**, it gets enqueued to run on a separate request while the workflow is suspended.
* If a step throws an error, like in `sendWelcomeEmail`, the step will automatically be retried until it succeeds (or hits a default max retry count).
* The workflow's `sleep` function allows us to suspend execution of the workflow without using up any resources. A sleep can be a few seconds, hours, days, or even months long.

<Callout>
We'll dive deeper into workflows, steps, retrying, and sleep in [Concepts](/docs/foundations/concepts).
</Callout>

## Create Your Route Handler

To invoke your new workflow, you'll have to add your workflow to a `POST` API Route Handler, `app/api/signup/route.ts`. Then, add the following code:

```typescript title="app/api/signup/route.ts"
import { userSignupWorkflow } from "@/workflows/user-signup";
import { NextResponse } from "next/server";
 
export async function POST(request: Request) {
 const { email } = await request.json();
 
 await userSignupWorkflow(email); // Executes asynchronously and doesn't block your app
 
 return NextResponse.json({
  message: "User signup workflow started",
 });
}
```

This Route Handler creates a `POST` request endpoint at `/api/signup` that will trigger your workflow.

<Callout>
Workflows can be triggered from API routes, Server Actions, or any server-side code.
</Callout>

## Run in Development

To start your development server, run the following command in your terminal in the Next.js root directory:

```bash
npm run dev
```

Once your development server is running, you can trigger your workflow by running this command in the terminal:

```bash
curl -X POST --json '{"email":"hello@example.com"}' http://localhost:3000/api/signup
```

This terminal command sends a `POST` request to your development server at `http://localhost:3000/api/signup` with the payload email `hello@example.com`, which will be consumed by your workflow function.

Watch the Next.js development server logs to see your workflow execute as well as the steps that are being processed.
